Integra todos los componentes del sistema y ejecuta el flujo completo: Prolog + Backend + Frontend funcionando en tiempo real.

🧠 ¿Qué lograrás en este paso?

En esta etapa final conectarás todo el ecosistema: generación de datos, servidor Prolog y la interfaz web interactiva.

  • 🔗 Integración completa del sistema
  • ⚡ Comunicación en tiempo real (Frontend ↔ Backend)
  • 📊 Visualización dinámica con gráficas
  • 🧠 Evaluación de lógica inteligente en acción
⚙️ 1 Ejecución del sistema
1️⃣ Iniciar Servidor
swipl server.pl

✔ Prolog levanta el servidor HTTP en el puerto 8080

2️⃣ Abrir Interfaz Web
Da clic en la consola el enlace del servidor activo: http://localhost:8080 o alternativamente desde un navegador pega este enlace.

✔ Interfaz lista para interactuar con el backend

🧭 2 Flujo de uso del sistema
📁 A. Carga del Dataset

El primer paso es alimentar el sistema con datos. Selecciona el archivo CSV generado y presiona Analizar.

  • ✔ Se envía al endpoint /upload
  • ✔ Prolog convierte el CSV en hechos
  • ✔ Se almacenan en memoria dinámica
💡 Este proceso equivale a "entrenar" el sistema lógico.
📊 B. Predicción y visualización

Ingresa valores en el formulario y presiona Consultar. El frontend envía datos a Prolog y recibe una respuesta en JSON.

  • ✔ Llamada a /predict
  • ✔ Cálculo del puntaje en Prolog
  • ✔ Render dinámico con Chart.js

⚡ La gráfica se actualiza sin recargar la página.

🧠 C. Transparencia del modelo

El sistema incluye un modal que explica la lógica utilizada para calcular el riesgo.

  • ✔ Explicación de fórmula
  • ✔ Interpretación del puntaje
  • ✔ Transparencia algorítmica

🎯 3. Reto: Ajuste del modelo

Ahora modificarás la lógica del sistema para cambiar el comportamiento del algoritmo.

Puntaje is D.edad*0.2 + D.fuma*35 + D.alcohol*15 - D.ejercicio*25.
Tu misión:
  • ✔ Reducir impacto del ejercicio a 10
  • ✔ Si edad > 60 → alcohol = 30
💡 Edita server.pl, guarda cambios y reinicia el servidor.
